# A basic overview of what the script does:

The script created is a log file analysis tool which reads log files and creates a structured report in a 
text file which displays the IP address of the attacker, the number of brute-force attempts, and the 
timestamp. This is done within a window of ten minutes, and at the end of the document, it 
displays the attacks again however, they are grouped together to see the total amount of attempts. 
Additionally, the script creates a visual graph and displays the top five IP addresses that had the 
most failed logins.
This is done, by using anchor points in each line to extract data that we want, excluding irrelevant 
data such as successful logins or other attacks.


## IP PARSE FUNCTION
Within the code, we use a parse function which extracts information from the log files by reading 
them line by line. To do this, we split up the line and set anchor points to allow the tool to extract 
relevant information such as the time stamp and the IP address. The function works by finding 
anchor points as mentioned above. The function reads the lines and stops when it finds the word 
“from” and extracts the following word, in this case it is our IP address. After the IP is found, we add 
an index point to move past the space and save the IP address as Ip.
Exception handling is used to prevent a crash if the item we are looking for is not present.
```python
def parse_auth_line(line):  
    parts = line.split()                                                #split line into different parts
    ts_str = " ".join(parts[0:3])                                       #joins first three parts together to make a timestamp
    try:
        ts = datetime.strptime(f"2025 {ts_str}", "%Y %b %d %H:%M:%S")   #assumeing year is 2025, make timestamp using ts extraced from line
    except Exception:                                                   #Exept, means that nothing will happen if there is nothing found
        ts = None


    ip = None
    event_type = "other"
    if "Failed password" in line:
        event_type = "failed"
    elif "Accepted password" in line or "Accepted publickey" in line:
        event_type = "accepted"
    if " from " in line:                                                # if from in line, we know there will be an ip address, from an ssh login
        try:                                                            #if from exists, get line immediatly after 'from'
            idx = parts.index("from")
            ip = parts[idx+1]
        except (ValueError, IndexError):                                #if there is no from, just return nothing to prevent program breaking
            ip = None
    return ts, ip, event_type                                           #return three extracted parts, or a truple if you want to be pedantic

#Important part, taken from Mark: Main function

```


## BRUTE FORCE DETECTION
This part of the code is responsible for deciding whether an entry in the log file was a brute force attempt or simply a failed login. 
The code checks if certain conditions are met, such as the time window of ten minutes or less and if there were five or 
more attempts from the same IP address. To do this, we sort out our timestamps in chronological 
order using a loop which traverses through each line and adds them to variable I, which represents 
our index position. Secondly, we declare ‘j’ and set it as the index position.
The parameter set is, that if j+1 is less than n and within the time frame of ten minutes, increase j 
and increase the counter. If the count is equal to or greater than five, append the set called 
incidents. The count is the window, which is the difference between I and j, plus 1.

```python
#Brute force detection:
incidents = []                                                          #currently empty list, will store attack information here
window = timedelta(minutes=10)                                          #setting a timeframe, 10 minutes

for ip, times in per_ip_timestamps.items():                             #loop through each ip and its timestamps
    times.sort()                                                        #sort timestamps in chronological order
    n = len(times)                                                      #variable to hold num of failed timestamps for current IP
    i = 0                                                               #loop counter, marks start point as index 0
    while i < n:                                                        #loop through each timestamp index, basically whatever number i is currently
        j = i                                                           #new variable with a terrible name cause ive no imagineation, this is what counts failed attemps, i is index, this is fails
        while j + 1 < n and (times[j+1] - times[i]) <= window:          #expand time frame or window from index postion i outwards, move j forward as long as i have another timestamp within range of 10mins
            j += 1                                                      #j ends with last timestamp within same time window starting at index point i
        count = j - i + 1                                               #how many failed logins in that window
        if count >= 5:                                                  #take data and put in list 
            incidents.append({
                "ip": ip,
                "count": count,
                "first": times[i].isoformat(),                          #i is index of first time, iso format makes timestamp readable
                "last": times[j].isoformat()                            #j is now last time
            })
# advance i past this cluster to avoid duplicate overlapping reports:
            i = j + 1                                                   #stops us from having segregated counts of the same ip
        else:
            i += 1                                                      #if fewer than 5 failed attempts move on
```
